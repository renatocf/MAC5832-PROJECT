Nuvem de palavras¶

Uma nuvem de palavras um tipo de representao visual de palavras que representa os termos mais frequentes de um texto com tamanhos diferentes. Essas nuvens podem ser usadas para identificar, rapidamente, que categorias de termos tm maior relevncia ou repetio numa anlise. Para este trabalho, sero utilizadas as bibliotecas wordcloud e matplotlib para gerar esse tipo de representao.

Palavras vazias¶

As Palavras vazias (em ingls, Stop words) so palavras comumente removidas de textos em processamento de linguagem natural. De modo geral, as listas de palavra vaziais incluem preposies, pronomes e artigos, que so essenciais para a constuo de frases, mas que acrescentam pouco ou nenhum significado semntivo s frases.

Com as nuvens de palavras, possvel confirmar o aparecimento das palavras vazias em tweets, verificando a sua relevncia quantitativa nas frases. Se as palavras vazias aparecerem muito, elas podem prejudicar o desempenho de um classificador de sentimentos. Como a prpria biblioteca wordcloud retira stop words automaticamente, criamos dois geradores: um sem filtro (raw_wordcloud) e outro sem (clean_wordcloud).

Filtragem de texto¶

Considerando uma aplicao que envolva processamento de texto, natural realizar filtragens com o objetivo de simplificar (e, assim, tornar mais relevante outras informaes) das palavras contidas nesse texto. Para este trabalho, foram tentadas 6 abordagens:

  • Sem filtro: o tweet original, no modificado
  • Com apenas caracteres alfanumricos: o tweet com apenas letras e nmeros ASCII
  • Sem menes a usurios: o tweet sem palavras comeadas por @
  • Sem hashtags: o tweet sem palavras comeadas por #
  • Com palavras lematizadas: o tweet com palavras transformadas para conter apenas seus radicais
  • Com distino entre palavras negadas: o tweet modificado com a juno de not e o seu termo sucessor, para distinguir a verso positiva e negativa de uma mesma palavra (por exemplo, happy e not happy).

# Regular expressions
import re

# NLTK
from nltk.stem import WordNetLemmatizer

def noop(tweet):
    return tweet

def remove_non_alpha(tweet):
    tweet = re.sub(r"[^a-zA-Z0-9]", ' ', tweet)
    return tweet

def remove_users(tweet):
    tweet = re.sub(r"@[^ ]+", ' ', tweet)
    return tweet

def remove_hashtags(tweet):
    tweet = re.sub(r"#[^ ]+", ' ', tweet)
    return tweet

lemmatizer = WordNetLemmatizer()
def lemmatize(tweet):
    words = tweet.split()
    words = [ lemmatizer.lemmatize(w) for w in words ]
    tweet = ''.join(words)
    return tweet
    
def negation(matchobj):
    if matchobj.group(1) == "not": return 'not-'
    elif matchobj.group(1) == "cannot": return 'can not-'
    elif matchobj.group(1) == "can't": return 'can not-'
    elif matchobj.group(1) == "won't": return 'will not-'
    elif matchobj.group(1) == "n't": return ' not-'
    else: return ' '

def negate(tweet):
    tweet = re.sub(r"(not|cannot|can't|won't|n't)\s+", negation, tweet)
    return tweet

def clean(tweet):
    return negate(tweet)

Treinamento¶

Para realizar o treinamento e decidir a melhor verso testada, utiliza-se a tcnica de validao cruzada para separar o conjunto de treinamento em um conjunto de treinamento menor e um conjunto de validao. O conjunto de validao pode ser aplicado para estimar a efetividade dos modelos, e para comparar o desempenho de cada um. Neste trabalho, a validao cruzada foi feita com 10 folds. Cerca de $1/10$ do conjunto de treinamento foi reservado, em cada rodada, para servir para a validao, enquanto o restante ficou para treinamento. A acurcia da predio sobre o conjunto de treinamento chamado de erro sobre a amostra (E_in). A acurcia da predio sobre o conjunto da validao o erro de validao (E_val). Espera-se que a mdia dos erros de validao seja uma boa estimativa do erro de predio do modelo. A diferena entre E_in e E_val (levando em conta mdias e desvio-padro) d uma estimativa da capacidade de generalizao do modelo escolhido.

Na construo do modelo, variaram-se quatro fatores: o tipo de extrao de caractersticas a filtragem dos dados (explicada na seo anterior), o modelo probabilstico utilizado na predio e a reduo de dimensionalidade do espao de caractersticas. Por simplicidade, o cdigo abaixo mostra apenas a verso final utilizada para a predio do conjunto de testes. Os resultados dos outros experimentos, bem como comentrios sobre os resultados, esto listados a seguir.

# NLTK
import nltk

X_train = training_set['text']
Y_train = training_set['airline_sentiment']

E_in = [0] * 10
E_val = [0] * 10

for idx, [train_index, test_index] in enumerate(KFold(X_train.size, n_folds=10)):
    print('Validation set #%d' % (idx+1))

    Xk_train, Xk_test = X_train[train_index], X_train[test_index]
    Yk_train, Yk_test = Y_train[train_index], Y_train[test_index]

    # Filtering list of words
    Xk_train_list = [ clean(w) for w in Xk_train.tolist() ]
    Xk_test_list  = [ clean(w) for w in Xk_test.tolist() ]

    bag = CountVectorizer(stop_words='english', tokenizer=nltk.word_tokenize)#, max_features=2500)
    bag.fit(Xk_train_list)

    test = bag.transform(Xk_test_list)

    logreg = LogisticRegression()
    logreg.fit(train, Yk_train)

    predK_test = logreg.predict(test)

    E_val[idx] = accuracy_score(Yk_test, predK_test) # logreg.score(bag_test, Yk_test)
    
    print('Training set accuracy : ', E_in[idx])
    print('Test set accuracy     : ', E_val[idx])
    print('')

print('E_in: ', np.average(E_in), '', np.std(E_in))
print('E_val: ', np.average(E_val), '', np.std(E_val))

Extração de características¶

Usar a frequncia das palavras como caracterstica para a classificao pode parecer, primeira vista, simplista. Entretanto, a anlise com as nuvens de palavras mostra que as palavras relevantes dos textos aparecem com maior frequncia em uma ou outra categoria de tweets. Por esse motivo, um modelo de caractersticas interessante o saco de palavras (bag of wods), que associa, a cada palavra de um dicionrio, a contagem do aparecimentos dessas palavras no texto. Outro modelo alternativo, muito usado para anlise de textos, o Tf-Idf, que pondera em um nico valor a frequncia de aparecimento e a raridade dos termos. Ambos os tipos de extrao de caractersticas foram testados nesse trabalho, obtendo-se os seguintes valores de E_in e E_val:

Tipo de extrao de caractersticas E_in E_val
Saco de palavras (CountVectorizer) 0.977 0.000 0.916 0.024
Tf-Idf (TfIdfVectorizer) 0.926 0.001 0.914 0.031

A acurcia da validao foi muito prxima em ambos os modelos, com um resultado ligeiramente pior para o Tf-Idf. Dada a natureza dos tweets e a classificao desejada (anlise de sentimentos) a informao de frequncia versus raridade agregada pelo Tf-Idf no apresenta grande vantagem para o modelo. Uma palavra classificada como positiva no sempre positiva somente porque ela apareceu apenas uma vez. Esse tipo de resultado mais ltil para buscadores, em que um texto um bom resultado de busca se ele contm as mesmas palavras raras que a pergunta feita.

Considerando os problemas obtidos com o Tf-Idf, o modelo de saco de palavras foi utilizado para o restante dos experimentos.

Em ambos os testes, utilizou-se a filtragem de stop words disponvel nos modelos CountVectorizer e TfIdfVectorizer do Scikit-Learning.

Obs: Os resultados da tabela foram obtidos usando a regresso logstica como modelo de predio e sem nenhuma filtragem.

Filtragem¶

Para tentar melhorar as contagens feitas no modelo de sacos de palavras, aplicamos as diferentes filtragens mostradas na seo anterior. Os resultados de E_in e E_val foram os seguintes:

Filtro E_in E_val
Sem filtro 0.977 0.000 0.916 0.024
Com apenas caracteres alfanumricos 0.974 0.000 0.909 0.028
Sem menes a usurios 0.976 0.000 0.914 0.024
Sem hashtags 0.975 0.000 0.915 0.025
Com palavras lematizadas 0.897 0.001 0.815 0.073
Com distino entre palavras negadas 0.979 0.000 0.915 0.023

interessante notar que a opo sem filtro foi melhor que quase todas as outras filtragens testadas. Isso pode ser explicado pelo resultado obtido quando deixaram-se apenas os caracteres alfanumricos: os caracteres "especiais", principalmente !, ? e emojis, devem agregar informaes para a classificao de sentimentos. As hashtags e menes no impactaram tanto na acurcia da validao, sendo as menes as mais relevantes. Esse fator pode ser explicado pela anlise feita nas nuvens de palavras: as companhias "JetAir" e "SouthWestAir", quase sempre positivas, devem ter impacto na predio. A lematizao foi desastrosa: tanto a acurcia na amostra quanto fora diminuram cerca de 10%. Como os tweets so curtos e as palavras tm grafia abreviada e incorreta (por conta da limitao de espao das mensagens e erros de digitao), a lematizao acabou gerando sufixos errados.

Dentre as filtragens, a nica que no foi mais ruim de forma conclusiva foi a que filtragem com distino de palavras negadas (acurcia mdia de 0.9156 para sem filtro contra 0.9147, e desvio-padra de aproximadamente 0.025 em ambas as medidas). Embora no houvesse uma vantagem clara em usar esse tipo de filtragem, ela foi mantido na verso final (funo clean), pois apresentou uma generalizao melhor no modelo final submetido no Kaggle.

Obs: Os resultados da tabela foram obtidos usando a regresso logstica como modelo de predio e o saco de palavras para extrao de caractersticas

Modelo probabilístico¶

Para fazer a classificao final a partir das contagens, foram testados dois modelos lineares simples: a regresso linear e a regresso logstica. Os valores de E_in e E_val obtidos doram os seguintes:

Modelo de predio E_in E_val
Regresso linear 0.998 0.000 0.674 0.018
Regresso logstica 0.979 0.000 0.915 0.023

A regresso logstica conseguiu se desempenhar muito melhor que a regresso linear - o que pode ser explicado porque a regresso logstica se adequa melhor a um problema de classificao (dando um valor que pode ser interpretado como a probabilidade do teste estar numa categoria ou outra) enquanto a regresso linear se enquadra melhor para dar a intensidade dessa classificao (sendo que valores mais distantes do zero so mais prximos de uma ou outra caracterstica).

Redução de dimensionalidade¶

O ltimo tipo de melhoria testada durante a criao do modelo foi a reduo da dimensionalidade do espao de caractersticas. A primeira possvel reduo limitar o tamanho das palavras consideradas no saco de palavras. Pelos experimentos, aps a retirada das palavras vazias, no h ganho em reduzir o nmero de palavras consideradas. De forma geral, porm, limitar o mximo para cerca de 2500 palavras no prejudicou muito a acurcia do modelo.

Outra forma de fazer uma reduo de dimensionalidade usando o PCA (principal component analysis). Nessa tcnica, realiza-se uma mudana de base que leva a base original para a base dos componenetes principais, que representam as principais direes de espalhamento em cada dimenso. O PCA pode ser calculado por meio da SVD (singular value decomposition), que fornece os vetores necessrios para a mudana de base. Embora essa mudana parecesse uma boa ideia, o custo de calcular a SVD para mais de 8000 dimenses (e mesmo para 2500). Por essa razo, no foi possvel fazer essa reduo de dimensionalidade.
